### Permission Bits

| Symbol | Value | Meaning on file | Meaning on directory |
|--------|:-----:|----------------|---------------------|
| `r` | 4 | Read file contents | List directory contents (`ls`) |
| `w` | 2 | Modify file | Create, delete, rename files inside |
| `x` | 1 | Execute file | Enter directory (`cd`) |
| `-` | 0 | Permission denied | — |

**Octal notation:** Each group (owner/group/other) is a sum: `rwx = 4+2+1 = 7`, `r-x = 5`, `r-- = 4`.

| Mode | Octal | Common use |
|------|:-----:|-----------|
| `rwxr-xr-x` | 755 | Executables, public directories |
| `rw-r--r--` | 644 | Regular files, configs |
| `rw-------` | 600 | Private keys, secrets |
| `rwx------` | 700 | Private scripts |
| `rwxrwxrwx` | 777 | ⚠️ World-writable — avoid |

## Linux — Commands

```bash
# Change permissions
chmod 755 script.sh             # Numeric: owner rwx, group r-x, other r-x
chmod +x script.sh              # Add execute for all
chmod -R 644 /var/www/html/     # Recursive
chmod go-w sensitive.txt        # Remove write from group and other

# Change ownership
chown alice file.txt            # Change owner
chown alice:devteam file.txt    # Change owner and group
chown -R alice:devteam /app/    # Recursive

# Special bits
chmod u+s /usr/bin/passwd       # Setuid: runs as file's owner (root), not the caller
chmod g+s /shared/dir           # Setgid: new files inherit directory's group
chmod +t /tmp                   # Sticky bit: only file owner can delete (used on /tmp)

# Find dangerous permissions
find / -perm -4000 -type f 2>/dev/null      # All setuid files
find / -perm -o+w -type f 2>/dev/null       # World-writable files (security risk)
```

### ACLs — Fine-Grained Control

Standard Unix permissions only have three levels. For more granularity, use ACLs:

```bash
# View ACL
getfacl file.txt

# Give a specific user read access without changing ownership
setfacl -m u:bob:r file.txt

# Give a group read+write
setfacl -m g:contractors:rw /project/

# Remove an ACL entry
setfacl -x u:bob file.txt

# Remove all ACLs
setfacl -b file.txt
```

## Windows — NTFS ACL Model

Windows uses **Access Control Lists (ACLs)** natively — each file has a list of Access Control Entries (ACEs), each specifying a user or group and their allowed/denied permissions.

Standard rights: **FullControl, Modify, ReadAndExecute, Read, Write**.

```powershell
# View current ACL
Get-Acl "C:\mydir" | Format-List

# Grant a user read-only access to a folder (including sub-items)
$acl  = Get-Acl "C:\mydir"
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    "DOMAIN\username",
    "Read",
    "ContainerInherit,ObjectInherit",   # Propagates to sub-folders and files
    "None",
    "Allow"
)
$acl.SetAccessRule($rule)
Set-Acl "C:\mydir" $acl

# icacls — simpler CLI alternative
icacls C:\mydir /grant "alice:(OI)(CI)RX"  # OI=object inherit, CI=container inherit
icacls C:\mydir /deny  "Guest:F"            # Deny full control to Guest
icacls C:\mydir /remove "alice"             # Remove alice's entry
icacls C:\mydir /reset /T                  # Reset all to inherited (recursive)

# Take ownership of an orphaned file (requires admin)
takeown /F C:\orphaned.txt
```

### Windows ACL Inheritance Flags

| Flag | Meaning |
|------|---------|
| `(OI)` | Object Inherit — applies to files within the folder |
| `(CI)` | Container Inherit — applies to sub-folders |
| `(NP)` | No Propagate — don't inherit further |
| `(IO)` | Inherit Only — doesn't apply to the folder itself |

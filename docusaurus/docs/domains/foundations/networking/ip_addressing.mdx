## IPv4 Address Structure

Every device on a network has an **IP address** — a unique identifier for routing.

**IPv4** addresses are 32 bits, written as 4 octets in decimal.

## CIDR Notation

**CIDR (Classless Inter-Domain Routing)** uses a `/prefix` to say how many leading bits are the **network** portion.

```
192.168.1.0/24

/24  → first 24 bits = network
     → last  8 bits  = host

Subnet mask : 255.255.255.0
Usable hosts: 2^8 − 2 = 254   (subtract network + broadcast)
Range       : 192.168.1.1 – 192.168.1.254
Broadcast   : 192.168.1.255
```

### Common CIDR Blocks

| CIDR | Subnet Mask | Usable Hosts | Typical Use |
|------|-------------|:------------:|-------------|
| `/8` | 255.0.0.0 | 16,777,214 | Large ISP allocations |
| `/16` | 255.255.0.0 | 65,534 | Large enterprise network |
| `/24` | 255.255.255.0 | 254 | Typical LAN segment |
| `/28` | 255.255.255.240 | 14 | Small office / cloud subnet |
| `/30` | 255.255.255.252 | 2 | Point-to-point links |
| `/32` | 255.255.255.255 | 1 | Single specific host (loopback / firewall rule) |

## Try It

<Tabs>
<TabItem value="linux" label="Linux">

```bash
# View your IP address and subnet mask
ip addr show

# Show your interface in CIDR notation
ip -br addr

# Calculate network info from an address
ipcalc 192.168.1.50/24
```

</TabItem>
<TabItem value="windows" label="Windows">

```powershell
# View all IP addresses and subnet masks
ipconfig /all

# PowerShell — see addresses with prefix length (CIDR equivalent)
Get-NetIPAddress -AddressFamily IPv4 | Format-Table InterfaceAlias, IPAddress, PrefixLength
```

</TabItem>
</Tabs>

:::tip[Try It]
Run the command for your OS. Find your machine's IP address and prefix length. Calculate the broadcast address and the number of usable hosts on your subnet. Then ping another device on the same network.
:::

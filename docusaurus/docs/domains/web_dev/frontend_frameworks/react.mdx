# React

> **Tool:** React · **Introduced:** 2013 (Facebook) · **Latest:** React 19 (2024) · **Status:** 🟢 Modern

React is a **UI library** for building component-based interfaces. It is intentionally minimal — it handles the view layer only. Routing, state management, and data fetching are handled by the ecosystem around it.

> **Prerequisite:** [JavaScript](../core/javascript) · [TypeScript](../typescript)
> **Courses:** [React Fundamentals](../../courses/react_fundamentals/index) · [React Advanced](../../courses/react_advanced/index)

## JSX

JSX is a syntax extension for JavaScript that lets you write HTML-like markup inside JavaScript. It is not required (you can use `React.createElement`), but it is universal in practice.

```jsx
// JSX is compiled to React.createElement() calls
const element = <h1 className="title">Hello, {name}!</h1>;
// Compiles to:
const element = React.createElement('h1', { className: 'title' }, `Hello, ${name}!`);

// JSX rules:
// 1. Return a single root element (use <> fragment if needed)
// 2. Close ALL tags — <br /> not <br>
// 3. Use camelCase for HTML attributes: className, htmlFor, onClick
// 4. Expressions in {curly braces}
// 5. Comments: {/* this is a comment */}

function Card({ title, children }) {
    return (
        <>  {/* Fragment — no extra DOM node */}
            <div className="card">
                <h2 className="card__title">{title}</h2>
                <div className="card__body">{children}</div>
            </div>
        </>
    );
}
```

## Components

React components are **JavaScript functions** that return JSX. The function name must be capitalised.

```tsx
// Functional component (modern standard)
interface ButtonProps {
    label: string;
    variant?: 'primary' | 'secondary' | 'danger';
    onClick: () => void;
    disabled?: boolean;
}

function Button({ label, variant = 'primary', onClick, disabled = false }: ButtonProps) {
    return (
        <button
            className={`btn btn--${variant}`}
            onClick={onClick}
            disabled={disabled}
            type="button"
        >
            {label}
        </button>
    );
}

// Using the component
<Button label="Save" variant="primary" onClick={handleSave} />
<Button label="Delete" variant="danger" onClick={handleDelete} disabled={isLoading} />
```

### Rendering Lists

```tsx
const posts = [
    { id: 1, title: 'Hello World' },
    { id: 2, title: 'React is fun' },
];

function PostList() {
    return (
        <ul>
            {posts.map(post => (
                // key must be stable, unique, and NOT the array index
                <li key={post.id}>
                    <a href={`/posts/${post.id}`}>{post.title}</a>
                </li>
            ))}
        </ul>
    );
}
```

### Conditional Rendering

```tsx
function Alert({ type, message }: { type: 'error' | 'success'; message: string }) {
    // Option 1: ternary (for if/else)
    return (
        <div className={type === 'error' ? 'alert alert--error' : 'alert alert--success'}>
            {message}
        </div>
    );
}

function Dashboard({ user }: { user: User | null }) {
    // Option 2: short-circuit && (for if-only)
    return (
        <main>
            {user && <WelcomeBanner name={user.name} />}
            {!user && <LoginPrompt />}
        </main>
    );
}
```

## Hooks

Hooks are functions that let functional components use React features. They must be called at the top level (not inside conditions or loops).

### `useState` — Local State

```tsx
import { useState } from 'react';

function Counter() {
    // [currentValue, setterFunction]
    const [count, setCount] = useState(0);

    // Functional update — use when new value depends on old value
    const increment = () => setCount(prev => prev + 1);
    const reset     = () => setCount(0);

    return (
        <div>
            <span>{count}</span>
            <button onClick={increment}>+</button>
            <button onClick={reset}>Reset</button>
        </div>
    );
}
```

### `useEffect` — Side Effects

```tsx
import { useEffect, useState } from 'react';

function UserProfile({ userId }: { userId: string }) {
    const [user, setUser] = useState<User | null>(null);

    useEffect(() => {
        // Effect runs after render when userId changes
        let cancelled = false;

        async function load() {
            const data = await fetchUser(userId);
            if (!cancelled) setUser(data);  // Avoid setting state after unmount
        }
        load();

        // Cleanup — runs before next effect or on unmount
        return () => { cancelled = true; };
    }, [userId]);  // Dependency array — only re-run if userId changes
    //  []       → run once on mount
    //  [a, b]   → run when a or b changes
    //  (no array) → run after every render (rarely what you want)

    if (!user) return <Spinner />;
    return <div>{user.name}</div>;
}
```

### `useRef` — Mutable Values and DOM Refs

```tsx
import { useRef, useEffect } from 'react';

function FocusInput() {
    // DOM ref — imperative access to a DOM element
    const inputRef = useRef<HTMLInputElement>(null);

    useEffect(() => {
        inputRef.current?.focus();  // Focus on mount
    }, []);

    return <input ref={inputRef} placeholder="Auto-focused" />;
}

// Mutable value — persists across renders, doesn't trigger re-render
function Timer() {
    const intervalRef = useRef<number | null>(null);

    const start = () => {
        intervalRef.current = window.setInterval(() => {
            console.log('tick');
        }, 1000);
    };

    const stop = () => {
        if (intervalRef.current) clearInterval(intervalRef.current);
    };

    return <><button onClick={start}>Start</button><button onClick={stop}>Stop</button></>;
}
```

### `useContext` — Shared Data Without Prop Drilling

```tsx
import { createContext, useContext, useState } from 'react';

// 1. Create the context
interface ThemeContextType {
    theme: 'light' | 'dark';
    toggle: () => void;
}
const ThemeContext = createContext<ThemeContextType | null>(null);

// 2. Provide it high in the tree
function App() {
    const [theme, setTheme] = useState<'light' | 'dark'>('light');

    return (
        <ThemeContext.Provider value={{ theme, toggle: () => setTheme(t => t === 'light' ? 'dark' : 'light') }}>
            <Router />
        </ThemeContext.Provider>
    );
}

// 3. Consume anywhere in the tree (no prop drilling)
function ThemeToggle() {
    const ctx = useContext(ThemeContext);
    if (!ctx) throw new Error('ThemeToggle must be used within ThemeContext.Provider');
    return <button onClick={ctx.toggle}>{ctx.theme} mode</button>;
}
```

### `useReducer` — Complex State Transitions

```tsx
import { useReducer } from 'react';

type State = { count: number; loading: boolean; error: string | null; };
type Action =
    | { type: 'increment' }
    | { type: 'decrement' }
    | { type: 'reset' };

function reducer(state: State, action: Action): State {
    switch (action.type) {
        case 'increment': return { ...state, count: state.count + 1 };
        case 'decrement': return { ...state, count: state.count - 1 };
        case 'reset':     return { ...state, count: 0 };
        default:          return state;
    }
}

function Counter() {
    const [state, dispatch] = useReducer(reducer, { count: 0, loading: false, error: null });

    return (
        <>
            <span>{state.count}</span>
            <button onClick={() => dispatch({ type: 'increment' })}>+</button>
            <button onClick={() => dispatch({ type: 'decrement' })}>-</button>
        </>
    );
}
```

### `useMemo` and `useCallback` — Performance

```tsx
import { useMemo, useCallback } from 'react';

function ExpensiveList({ items, filter }: { items: Item[]; filter: string }) {
    // Only recompute when items or filter change
    const filtered = useMemo(
        () => items.filter(item => item.name.includes(filter)),
        [items, filter]
    );

    // Stable function reference — child won't re-render unnecessarily
    const handleClick = useCallback((id: string) => {
        console.log('Clicked:', id);
    }, []); // No dependencies — stable across all renders

    return (
        <ul>
            {filtered.map(item => (
                <ListItem key={item.id} item={item} onClick={handleClick} />
            ))}
        </ul>
    );
}
```

### Custom Hooks

Extract reusable logic into a custom hook (must start with `use`):

```tsx
// useLocalStorage.ts
function useLocalStorage<T>(key: string, initialValue: T) {
    const [value, setValue] = useState<T>(() => {
        try {
            const item = window.localStorage.getItem(key);
            return item ? JSON.parse(item) : initialValue;
        } catch {
            return initialValue;
        }
    });

    const setStoredValue = (newValue: T) => {
        setValue(newValue);
        window.localStorage.setItem(key, JSON.stringify(newValue));
    };

    return [value, setStoredValue] as const;
}

// Usage
const [theme, setTheme] = useLocalStorage('theme', 'light');
```

## State Management Ecosystem

```
useState       — local, simple value
useReducer     — local, complex transitions (multiple fields, computed state)
Context        — shared within a subtree (theme, auth, language)
Zustand        — lightweight global store (recommended for most apps)
Redux Toolkit  — large-scale apps with many developers
TanStack Query — server state: caching, background refresh, optimistic updates
Jotai / Recoil — atomic state (fine-grained)
```

### Zustand

```tsx
import { create } from 'zustand';

interface CartStore {
    items: CartItem[];
    addItem: (item: CartItem) => void;
    removeItem: (id: string) => void;
    total: () => number;
}

const useCartStore = create<CartStore>((set, get) => ({
    items: [],
    addItem: (item) => set(state => ({ items: [...state.items, item] })),
    removeItem: (id) => set(state => ({ items: state.items.filter(i => i.id !== id) })),
    total: () => get().items.reduce((sum, item) => sum + item.price, 0),
}));

// In any component — no Provider needed
function Cart() {
    const { items, removeItem, total } = useCartStore();
    return (
        <aside>
            {items.map(item => (
                <div key={item.id}>
                    {item.name}
                    <button onClick={() => removeItem(item.id)}>Remove</button>
                </div>
            ))}
            <p>Total: ${total()}</p>
        </aside>
    );
}
```

### TanStack Query

```tsx
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';

function UserList() {
    // Automatic caching, background refetching, loading/error states
    const { data: users, isLoading, error } = useQuery({
        queryKey: ['users'],
        queryFn: () => fetch('/api/users').then(r => r.json()),
        staleTime: 5 * 60 * 1000,  // Cache fresh for 5 minutes
    });

    const queryClient = useQueryClient();
    const deleteMutation = useMutation({
        mutationFn: (id: string) => fetch(`/api/users/${id}`, { method: 'DELETE' }),
        onSuccess: () => {
            queryClient.invalidateQueries({ queryKey: ['users'] }); // Re-fetch
        },
    });

    if (isLoading) return <Spinner />;
    if (error) return <ErrorMessage error={error} />;

    return (
        <ul>
            {users.map(user => (
                <li key={user.id}>
                    {user.name}
                    <button onClick={() => deleteMutation.mutate(user.id)}>Delete</button>
                </li>
            ))}
        </ul>
    );
}
```

function App() {
    return (
        <BrowserRouter>

## Performance Patterns

### React.memo — Skip Re-renders

```tsx
import { memo } from 'react';

// Only re-renders if props change (shallow comparison)
const ArticleCard = memo(function ArticleCard({ article, onLike }: ArticleCardProps) {
    return (
        <article>
            <h2>{article.title}</h2>
            <button onClick={() => onLike(article.id)}>Like</button>
        </article>
    );
});
```

### Lazy Loading with `Suspense`

```tsx
import { lazy, Suspense } from 'react';

// Code-split — only downloads when this route is visited
const Dashboard = lazy(() => import('./pages/Dashboard'));
const Settings  = lazy(() => import('./pages/Settings'));

function App() {
    return (
        <Suspense fallback={<LoadingSpinner />}>
            <Routes>
                <Route path="/dashboard" element={<Dashboard />} />
                <Route path="/settings" element={<Settings />} />
            </Routes>
        </Suspense>
    );
}
```

## React 19 — New Features

React 19 (2024) introduced:

- **Server Components (stable):** Components that render on the server with no client-side JS — zero bundle cost.
- **`use()` hook:** Read a Promise or Context in render: `const data = use(fetchDataPromise)`.
- **Actions:** Built-in form handling with `useActionState` — no `onSubmit` boilerplate.
- **Document Metadata:** `<title>`, `<meta>`, `<link>` can be rendered anywhere in the tree.

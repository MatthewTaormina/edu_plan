## Execution Context and The Event Loop

JavaScript is **single-threaded** — it has one call stack and executes one thing at a time. To avoid blocking the UI on slow operations (network requests, timers, file reads), it uses an **event loop**.

```
              JavaScript Runtime

 ┌──────────────────┐    ┌────────────────────┐
 │    Call Stack    │    │     Web APIs        │
 │                  │    │  (provided by       │
 │  execute()       │───►│   the browser)      │
 │  greet()         │    │  - setTimeout       │
 │  main()          │    │  - fetch            │
 └────────┬─────────┘    │  - DOM Events       │
          │              └──────────┬───────────┘
          │                        │  when done
          │                        ▼
          │              ┌────────────────────┐
          │              │   Task Queue       │
          │◄─────────────│   (callbacks)      │
          │  event loop  └────────────────────┘
          │  checks when
          │  stack empty ┌────────────────────┐
          │◄─────────────│  Microtask Queue   │
                         │  (Promise.then,    │
                         │   queueMicrotask)  │
                         └────────────────────┘
```

1. JS executes synchronous code on the **Call Stack**
2. Async operations (e.g. `setTimeout`, `fetch`) are handed to **Web APIs**
3. When complete, their callbacks go to the **Task Queue** (or Microtask Queue for Promises)
4. The **Event Loop** moves callbacks onto the Call Stack only when it's empty
5. Microtasks (Promises) drain completely before the next Task Queue item runs

```javascript
console.log('1');              // Runs immediately (call stack)

setTimeout(() => {
    console.log('3');          // Macro-task — runs after microtasks clear
}, 0);

Promise.resolve().then(() => {
    console.log('2');          // Microtask — runs before setTimeout callback
});

console.log('4');              // Runs immediately (call stack)

// Output: 1, 4, 2, 3
```

:::note
`setTimeout(fn, 0)` doesn't mean "run immediately" — it means "run as soon as the call stack is empty and after all microtasks have resolved."
:::

## The Prototype Chain

JavaScript uses **prototypal inheritance**, not classical inheritance. Every object has an internal `[[Prototype]]` link to another object. When you access a property, JS walks the chain until it finds it or reaches `null`.

```javascript
// ── Legacy syntax: prototype manipulation directly ────────────────
function Animal(name) {
    this.name = name;
}
Animal.prototype.speak = function () {
    console.log(`${this.name} makes a noise.`);
};

const dog = new Animal('Rex');
dog.speak();                    // Found on Animal.prototype
dog.hasOwnProperty('name');    // Found on dog itself
dog.toString();                // Found on Object.prototype

// ── Modern syntax: ES6 class (syntactic sugar over the above) ─────
class Animal {
    constructor(name) {
        this.name = name;
    }
    speak() {
        console.log(`${this.name} makes a noise.`);
    }
}

class Dog extends Animal {
    speak() {
        console.log(`${this.name} barks.`);
    }
}

const d = new Dog('Rex');
d.speak();              // Dog.prototype.speak (overrides Animal)
Object.getPrototypeOf(d) === Dog.prototype;  // true
```

:::tip
`class` syntax is preferred in modern code. The prototype chain still exists under the hood — understanding it helps when debugging `this` binding issues or reading older codebases.
:::

## `this` Binding

`this` in JavaScript is determined by **how a function is called**, not where it's defined (with one exception: arrow functions).

```javascript
const obj = {
    name: 'Widget',

    // Regular function — 'this' = whoever called it
    getName() {
        return this.name;
    },

    // Arrow function — 'this' is inherited from surrounding scope
    getNameArrow: () => {
        return this.name; // 'this' is the outer scope, NOT obj
    }
};

obj.getName();      // 'Widget' — called on obj, so this = obj
obj.getNameArrow(); // undefined — arrow function, this = outer scope

// Common pitfall: losing 'this' when passing a method as a callback
const get = obj.getName;
get();              // undefined (or error in strict mode) — no longer called on obj

// Fix 1: bind()
const bound = obj.getName.bind(obj);
bound();            // 'Widget'

// Fix 2: arrow function in the callback
setTimeout(() => obj.getName(), 100); // 'Widget' — still called on obj
```

## The DOM API

The Document Object Model is a live, tree-structured representation of the HTML document. JavaScript reads and writes it to update the page without a reload.

### Selecting Elements

```javascript
// Single element
const header  = document.getElementById('main-header');
const button  = document.querySelector('.btn-submit');        // First match
const title   = document.querySelector('h1');

// Collections (live NodeList or HTMLCollection)
const allCards = document.querySelectorAll('.card');           // Static NodeList
const items    = document.getElementsByClassName('item');      // Live HTMLCollection

// Traversal
const parent   = element.parentElement;
const children = element.children;
const next     = element.nextElementSibling;
const prev     = element.previousElementSibling;
```

### Modifying the DOM

```javascript
// Content
element.textContent = 'Hello';           // Text only (safe)
element.innerHTML   = '<b>Hello</b>';    // Parses HTML (XSS risk — sanitise first)

// Attributes
element.setAttribute('aria-expanded', 'true');
element.getAttribute('href');
element.removeAttribute('disabled');
element.toggleAttribute('hidden');

// Classes
element.classList.add('active', 'visible');
element.classList.remove('hidden');
element.classList.toggle('open');
element.classList.replace('old', 'new');
element.classList.contains('active');    // true/false

// Styles (prefer classes; use for dynamic values)
element.style.transform = 'translateY(-4px)';
element.style.setProperty('--custom-var', '42px');

// Create and insert nodes
const card = document.createElement('div');
card.className = 'card';
card.textContent = 'New card';

parent.appendChild(card);               // Add at end
parent.prepend(card);                   // Add at start
parent.insertBefore(card, sibling);     // Insert before specific child
reference.after(card);                  // Insert after element
reference.before(card);                 // Insert before element
element.replaceWith(card);              // Replace element
element.remove();                       // Remove element
```

### Events

```javascript
// Add event listener
const button = document.querySelector('#submit-btn');

button.addEventListener('click', function(event) {
    event.preventDefault();      // Stop form submission / default link navigation
    event.stopPropagation();     // Stop event from bubbling to parent elements
    console.log('Clicked!', event.target);
});

// Remove event listener (must use same function reference)
function handleClick(e) { console.log(e); }
button.addEventListener('click', handleClick);
button.removeEventListener('click', handleClick);

// Event options
button.addEventListener('click', handleClick, {
    once:    true,   // Automatically remove after first call
    passive: true,   // Promise not to call preventDefault (improves scroll perf)
    capture: true,   // Run in capture phase instead of bubble phase
});
```

### Event Propagation

Events travel in two phases: **capturing** (down the tree) then **bubbling** (back up):

```
Document → html → body → main → section → button   (capture phase)
button → section → main → body → html → Document   (bubble phase)
```

**Event delegation** — attach one listener to a parent to handle many children:

```javascript
// Instead of adding a listener to every .card:
document.querySelector('.card-grid').addEventListener('click', (event) => {
    const card = event.target.closest('.card');  // Walk up to find the card
    if (!card) return;                           // Click was on empty space
    console.log('Card clicked:', card.dataset.id);
});

// Works for dynamically added cards too — no re-registration needed
```

## Asynchronous JavaScript

### Promises

A Promise represents the eventual completion (or failure) of an async operation.

```javascript
// Creating a Promise
const delay = (ms) => new Promise((resolve, reject) => {
    if (ms < 0) reject(new Error('Delay must be positive'));
    setTimeout(() => resolve(`Done after ${ms}ms`), ms);
});

// Consuming with .then()/.catch()
delay(1000)
    .then(result => console.log(result))
    .catch(error => console.error(error))
    .finally(() => console.log('Always runs'));

// Promise combinators
await Promise.all([fetch('/a'), fetch('/b')]);        // Wait for ALL — fail fast
await Promise.allSettled([fetch('/a'), fetch('/b')]); // Wait for ALL — no fail fast
await Promise.race([fetch('/a'), fetch('/b')]);       // First to settle wins
await Promise.any([fetch('/a'), fetch('/b')]);        // First to SUCCEED wins
```

### Async / Await

`async/await` is syntactic sugar over Promises — it makes async code read like synchronous code.

```javascript
async function getUserData(userId) {
    try {
        const response = await fetch(`https://api.example.com/users/${userId}`);

        if (!response.ok) {
            throw new Error(`HTTP ${response.status}: ${response.statusText}`);
        }

        const data = await response.json();
        return data;
    } catch (error) {
        console.error('Request failed:', error);
        throw error; // Re-throw so the caller can handle it
    }
}

// Parallel requests — don't await inside a loop
async function loadAll(ids) {
    // ❌ Sequential — waits for each before starting the next
    for (const id of ids) {
        const user = await getUserData(id); // One at a time
    }

    // ✅ Parallel — all requests fire at once
    const users = await Promise.all(ids.map(id => getUserData(id)));
}
```

### Fetch API

```javascript
// GET request
const response = await fetch('/api/posts');
const posts = await response.json();

// POST with JSON body
const response = await fetch('/api/posts', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({ title: 'Hello', content: 'World' })
});

// Error handling — fetch only rejects on network failure, not HTTP errors
if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message);
}

// Abort a request
const controller = new AbortController();
setTimeout(() => controller.abort(), 5000);  // Cancel after 5s

const response = await fetch('/api/slow', {
    signal: controller.signal
});
```

## ES Modules

The modern way to split code into reusable files. Supported natively in all modern browsers and Node.js (v12+).

```javascript
// math.js — named exports
export function add(a, b) { return a + b; }
export function subtract(a, b) { return a - b; }
export const PI = 3.14159;

// utils.js — default export (one per file)
export default function formatCurrency(amount, currency = 'USD') {
    return new Intl.NumberFormat('en-US', { style: 'currency', currency }).format(amount);
}

// main.js — imports
import formatCurrency from './utils.js';          // Default import
import { add, PI } from './math.js';              // Named imports
import { add as sum } from './math.js';           // Renamed import
import * as MathUtils from './math.js';           // Namespace import

// Dynamic import — load a module on demand
const { heavy } = await import('./heavy-module.js');

// In HTML
// <script type="module" src="main.js"></script>
```

## ES6+ Language Features

### Destructuring

```javascript
// Array destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];
// first=1, second=2, rest=[3,4,5]

// Object destructuring
const { name, age, role = 'learner' } = user;   // With default
const { name: userName } = user;                 // Rename

// In function parameters
function greet({ name, age = 0 }) {
    return `Hello ${name}, age ${age}`;
}
```

### Spread and Rest

```javascript
// Spread — expand into individual elements
const merged = { ...defaults, ...overrides };      // Merge objects
const copy   = [...original, newItem];             // New array with extra item
Math.max(...[1, 2, 3]);                            // Spread into function args

// Rest — collect remaining arguments
function sum(first, ...others) {                   // others = array
    return others.reduce((acc, n) => acc + n, first);
}
```

### Template Literals

```javascript
const name = 'World';

// Multi-line strings
const html = `
  <div class="card">
    <h2>${name}</h2>
    <p>${new Date().toLocaleDateString()}</p>
  </div>
`;

// Tagged templates (used by libraries like styled-components, graphql-tag)
const query = gql`
  query GetUser($id: ID!) {
    user(id: $id) { name email }
  }
`;
```

### Optional Chaining and Nullish Coalescing

```javascript
// Optional chaining — safe property access
const city = user?.address?.city;          // undefined if any part is null/undefined
const first = arr?.[0];                    // Safe array access
const result = fn?.();                     // Safe function call

// Nullish coalescing — only uses right side if left is null/undefined (not falsy)
const port = process.env.PORT ?? 3000;     // 3000 only if PORT is null/undefined
const name = user.name ?? 'Anonymous';    // NOT overridden by empty string ""
```

### Symbols, iterators, and generators

```javascript
// Symbol — unique, non-string property key
const id = Symbol('id');
obj[id] = 123;  // Not enumerable, can't clash with string keys

// Generator — pauseable function
function* range(start, end) {
    for (let i = start; i <= end; i++) {
        yield i;  // Pause, return value, resume on next()
    }
}
for (const n of range(1, 5)) {
    console.log(n); // 1 2 3 4 5
}
```

## WeakMap and WeakRef

```javascript
// WeakMap — keys must be objects; doesn't prevent garbage collection
const cache = new WeakMap();

function process(element) {
    if (cache.has(element)) return cache.get(element);
    const result = expensiveComputation(element);
    cache.set(element, result);    // When element is GC'd, cache entry disappears
    return result;
}

// WeakRef — hold a reference that doesn't prevent GC
const weakRef = new WeakRef(heavyObject);
const obj = weakRef.deref();  // undefined if garbage-collected
```
